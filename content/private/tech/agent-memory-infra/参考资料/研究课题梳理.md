# Agent Memory 系统研究课题梳理

基于版本B架构图和 [[tech/agent-mem-papers/]] 论文调研，将各模块按**研究投入需求**分类整理。

---

## 一、需要深入研究的算法/策略课题

### 🔴 高优先级（核心差异化能力）

| 课题 | 所属模块 | 研究内容 | 参考论文/方向 | 投入评估 |
|------|----------|----------|---------------|----------|
| **压缩遗忘算法** | Memory Processing | 记忆何时压缩、如何压缩、何时遗忘的策略 | [[working-memory.md\|AgentFold]], [[working-memory.md\|ACON]], MemGPT的遗忘策略 | ⭐⭐⭐ 需专门算法工程师 |
| **质量评估方法** | Memory Processing | 记忆质量打分模型、置信度评估、去重策略 | [[experiential-memory.md\|Reflexion]] 的语言反馈机制 | ⭐⭐⭐ 需结合业务场景研究 |
| **Skill挖掘算法** | Skill Mining | 从执行轨迹中提取可复用技能、技能泛化 | [[experiential-memory.md\|SkillWeaver]], [[experiential-memory.md\|Agent Workflow Memory]] | ⭐⭐⭐ 核心研究课题 |
| **混合检索排序** | Memory API | 相关性+新近性+重要性的融合排序策略 | [[斯坦福-generative-agents-notes.md\|斯坦福AI小镇]] 的三因子评分 | ⭐⭐ 有成熟方案需调优 |
| **Community聚类优化** | Semantic Memory | Leiden算法参数调优、动态社区发现 | GraphRAG社区聚类实践 | ⭐⭐ 算法已有，需适配场景 |

### 🟡 中优先级（增强能力）

| 课题 | 所属模块 | 研究内容 | 参考方向 | 投入评估 |
|------|----------|----------|----------|----------|
| **多路检索融合** | Memory API | 4层记忆的检索结果融合与重排序 | [[working-memory.md\|MemoRAG]] | ⭐⭐ 工程+策略研究 |
| **技能版本管理** | Skill Mining | 技能演进、冲突解决、回滚机制 | [[experiential-memory.md\|Remember Me, Refine Me]] | ⭐⭐ 偏工程策略 |
| **反思触发策略** | Memory Processing | 何时触发反思、反思粒度控制 | [[斯坦福-generative-agents-notes.md\|反思机制]] | ⭐⭐ 策略研究 |
| **Episode生命周期** | Episode Memory | 情景记忆的衰减、归档、再激活策略 | [[experiential-memory.md\|MemRL]] | ⭐⭐ 需结合实际场景 |
| **智能推荐算法** | Skill Mining | 基于上下文的技能推荐、冷启动策略 | 推荐系统+Agent场景结合 | ⭐⭐ 推荐算法迁移 |

## 二、设计与工程实现课题（偏架构设计）

### 🟢 设计为主（方案选型+接口定义）

| 课题 | 所属模块 | 设计要点 | 复杂度 |
|------|----------|----------|--------|

| **统一访问层抽象** | Memory API | 跨层检索的统一封装、事务管理 | 中等 |

| **Episode Memory实现** | 存储层 | Milvus向量检索 | 标准方案 |
| **Procedural Memory实现** | 存储层 | PostgreSQL关系存储 | 标准方案 |

## 四、按人力配置建议

### 算法研究团队（2-3人）

**核心职责**：
1. **压缩遗忘算法** - 研究记忆压缩策略，平衡信息保留与存储成本
2. **质量评估方法** - 设计记忆质量评估模型，支持自动整理决策
3. **Skill挖掘算法** - 从执行数据中提取可复用技能，构建技能库
4. **混合检索排序** - 优化多因子排序策略，提升检索效果

**参考论文清单**：
- [[working-memory.md]]（上下文压缩）
- [[experiential-memory.md]]（技能学习）
- [[斯坦福-generative-agents-notes.md]]（检索策略）

### 架构设计团队（2人）

**核心职责**：
1. **语义记忆架构** - 设计Phrase Node提取、Community聚类、摘要生成流程
2. **记忆-推理协同** - 设计检索与规划的交互机制
3. **多Agent记忆共享** - 设计分布式记忆同步与权限模型
4. **存储层Schema** - 设计4层存储的数据模型与访问接口

**参考方案**：
- GraphRAG开源实现
- [[0-agent-memory-overview.md]]（整体架构）

### 工程开发团队（4-5人）

**核心职责**：
1. **核心API实现** - Memory API接口开发与性能优化
2. **存储层集成** - Redis/Milvus/PostgreSQL/Neo4j集成
3. **智能处理模块** - Memory Processing Pipeline实现
4. **基础设施搭建** - K8s部署、监控、消息队列
5. **Agent Framework适配** - LangGraph集成与调用优化

## 五、研究课题依赖关系

```
第一层（基础）：
├── 存储层Schema设计
├── Memory API接口设计
└── 统一访问层抽象

第二层（核心算法）：
├── 压缩遗忘算法 ← 依赖质量评估方法
├── 混合检索排序
└── Community聚类优化

第三层（高级特性）：
├── Skill挖掘算法 ← 依赖Episode存储实现
├── 智能推荐算法 ← 依赖Skill库
├── 语义记忆架构 ← 依赖Community聚类
└── 树形摘要轴设计 ← 依赖质量评估

第四层（系统级）：
├── 记忆-推理协同 ← 依赖第三层全部
├── 多Agent记忆共享 ← 依赖语义记忆架构
└── 自进化记忆系统 ← 依赖全部前置
```

## 六、近期（3个月）研究重点建议

### Phase 1：基础验证（Month 1）

1. **压缩遗忘算法调研**
   - 研究MemGPT、AgentFold的压缩策略
   - 设计适合业务场景的压缩触发条件
   - 输出：压缩策略设计文档

2. **混合检索排序原型**
   - 实现相关性+新近性+重要性的基础融合
   - 在测试数据上验证效果
   - 输出：排序算法原型

3. **Semantic Memory架构设计**
   - 确定Phrase Node提取方案（LLM-based or 规则）
   - 设计Community摘要生成流程
   - 输出：架构设计文档

### Phase 2：核心实现（Month 2-3）

1. **Skill挖掘算法研究**
   - 分析执行轨迹特征
   - 设计技能提取和泛化策略
   - 输出：技能挖掘模块设计

2. **质量评估方法设计**
   - 定义记忆质量维度
   - 设计自动评分模型
   - 输出：质量评估API设计

3. **集成验证**
   - 各模块联调
   - 端到端场景验证
   - 输出：集成测试报告
